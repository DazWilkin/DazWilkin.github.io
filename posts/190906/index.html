September 6, 2019

Now that I’ve (p)retired from Google, I’m starting this blog and will no longer post stories to Medium.

As I concluded my time at Google, I wrapped up work on a Trillian prototype. As it remains Google’s IP, I’m not permitted to discuss it here.

I’ve begun work on another Trillian prototype for Python package transparency, informally pypi-transparency.

The goal of pypi-transparency is very similar to the underlying motivation for the Golang team’s Checksum Database (also built with Trillian). Even though, PyPi provides hashes of the content of packages it hosts, the developer must trust that PyPi’s data is consistent. One ambition with pypi-transparency is to provide a companion, tamperproof log of PyPi package files in order to provide a double-check of these hashes.

Currently pypi-transparency depends upon a local cache of Python packages to determine correct filenames.

For example, the following requirements.txt:

grpcio==1.23.0
opencensus==0.7.2
protobuf==3.9.1

Corresponds, for my Python3 on Linux x64 to the following files:

grpcio-1.23.0-cp37-cp37m-manylinux1_x86_64.whl
opencensus-0.7.2-py2.py3-none-any.whl
protobuf-3.9.1-cp37-cp37m-manylinux1_x86_64.whl

There are many more flavors: not only wheels but sources, and variants for different OSs etc.

And these correspond to the following PyPi URLs:

https://files.pythonhosted.org/packages/d6/c3/65db90ec27181edf491c26aa998ae631e50cd1f04ee8d8d513a95e3937f3/grpcio-1.23.0-cp27-cp27mu-manylinux1_x86_64.whl
https://files.pythonhosted.org/packages/b6/13/c37904d9f77c320dfd39b6d5c6bd95947c59e19baa3c6c4f4df6418d1433/opencensus-0.7.2-py2.py3-none-any.whl
https://files.pythonhosted.org/packages/c7/60/19c2c3b563c8a5ebbc5f17982fd794f415cfc4633a8248ab3e23a47662bc/protobuf-3.9.1-cp27-cp27mu-manylinux1_x86_64.whl

So, it’s non-trivial (and thus avoided) for pypi-transparency to convert package name*versions into filenames and URLs. Instead pypi-transparency leverages pip download to do this heavy-lifting and then uses cached files for its processing.

From the filenames (whl, zip etc.), it’s possible to determine the PyPi URL. The URL is https://files.pythonhosted.org/packages/ followed by a BLAKE2b hash of the package’s contents, followed by the filename. The BLAKE2b has is 64-hex digits and PyPi shards this into 2x2-digits subdirectories followed by the remaining 60 (60+2+2==64) digits followed by the filename. The Blake2b hash of the grpcio v1.23.0 package (grpcio-1.23.0-cp27-cp27mu-manylinux1_x86_64.whl) for my machine is d6c365db90ec27181edf491c26aa998ae631e50cd1f04ee8d8d513a95e3937f3 and so this becomes d6/c3/65...
